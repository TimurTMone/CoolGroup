Модуль 1 — Kanban-конвейер · 5 этапов цеха
#ТребованиеПриоритет
1Визуальная Kanban-доска с колонками-этапами: 1.Заказ → 2.Крой → 3.Пошив → 4.Проверка/ОТК → 5.Готово. Количество заказов и прогресс задач на каждом этапе.Высокий
2Drag-and-drop перемещение заказа между этапами — с автоматическим push всем участникам этапаВысокий
3WIP-лимиты (Work In Progress): максимальное кол-во заказов на этапе — основа теории ограничений. При превышении — алерт менеджеру.Средний
4Цветовая маркировка карточек: 🟢 в срок · 🟡 риск просрочки · 🔴 просрочен · ⚪ новыйСредний
5AI-индикатор «узкого места» (bottleneck): этап с наибольшей задержкой выделяется автоматическиСредний
6Настройка этапов администратором: название, порядок, WIP-лимит, добавление/удаление этапа без перепрограммированияСредний
7Автоприём заказа из B2B-контура: когда B2B-заказ переходит в стадию «Производство» — заказ на Kanban создаётся автоматически со всеми параметрамиВысокий
8Обратная синхронизация: прогресс Kanban (Крой → Пошив → Проверка → Готово) отображается в B2B-карточке — менеджер видит без звонков в цехВысокий

Модуль 3 — Задачи рабочих (мобильное приложение для каждого сотрудника цеха)
#ТребованиеПриоритет
1Создание задачи с привязкой к заказу и этапу Kanban-доски (кройка / пошив / ОТК)Высокий
2Назначение задачи конкретному рабочему (швея, закройщик, контролёр) с дедлайномВысокий
3Статусы: Ожидание → В работе → Завершено. Смена статуса рабочим — 2–3 нажатия, не больше. Максимальная простота.Высокий
4Упрощённый UI для рабочего: только его задачи, крупные кнопки, минимум текста. Адаптировано под человека у швейной машины.Высокий
5Push рабочему при назначении новой задачи; push менеджеру при завершении задачи рабочимВысокий
6Фото результата работы: рабочий делает фото готового изделия/операции прямо из приложенияСредний
7Временные метки: автоматическая фиксация времени начала и завершения задачи — основа цифровых следовСредний
8Авто-генерация задач при создании заказа по шаблонам типа изделия — менеджер не создаёт задачи вручнуюСредний
Модуль 4 — Встроенный real-time чат (ключевое отличие от всех существующих решений)
Главная боль существующих Kanban-систем: коммуникация ведётся в WhatsApp/Telegram — контекст теряется, обсуждение дефекта по заказу №142 рассыпано по разным чатам. Cool Group решает это встроенным чатом, привязанным к заказам и задачам.
#ТребованиеПриоритет
1Real-time личные сообщения (WebSocket): менеджер ↔ рабочий, директор ↔ менеджер — без WhatsAppВысокий
2Комментарии к заказу: обсуждение дефекта, уточнение лекала — всё хранится в контексте конкретного заказаВысокий
3Комментарии к задаче: вопрос рабочего по операции — ответ менеджера прямо в задаче, не в отдельном чатеВысокий
4Отправка фото/файлов в чат: фото дефекта, образца, лекала — с привязкой к заказуВысокий
5Статус прочтения сообщений, push о новых сообщенияхВысокий
6Справочник сотрудников с быстрым поиском по роли, отделу, имени для начала диалогаСредний
